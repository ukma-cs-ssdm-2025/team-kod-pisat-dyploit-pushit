
# Final Project Report — Flick.ly

## Team team-kod-pisat-dyploit-pushit

## Executive Summary

**Flick.ly** — це повноцінна веб-платформа для оцінювання фільмів, написання рецензій, отримання персональних рекомендацій та взаємодії з іншими користувачами.
Проєкт реалізовано в межах курсу *Software Systems Development Methods* у НаУКМА.

Сервіс дозволяє користувачам знаходити нові фільми, зберігати їх у “Favorites”, слідкувати за активністю друзів, читати їхні рецензії й ділитися власними враженнями.


## Technology Stack

* **Backend:** Node.js + Express + PostgreSQL
* **Frontend:** React.js
* **Authorization:** JWT
* **Testing:** Jest (Unit + Integration)
* **Documentation:** OpenAPI / Swagger + Markdown documentation


## System Functionality

Платформа забезпечує можливість:

* Писати рецензії до фільмів та ставити оцінки
* Переглядати детальні сторінки фільмів із рейтингами
* Додавати фільми в особистий список *Favorites*
* Шукати та додавати інших користувачів у друзі
* Отримувати рекомендації на основі історії оцінок та вподобань друзів
* Переглядати активність друзів (їхні рецензії, оцінки, Favorites)

Адміністратор має можливість керувати базою фільмів і модерувати контент, написаний користувачами.

## 4. Implemented Modules and Features

### 4.1. Movie Management Module

**Функції:**

* Перегляд детальної інформації про фільм
* Пошук та фільтрація
* Сторінка рейтингу та рецензій
* Середній рейтинг на основі всіх оцінок користувачів


### 4.2. Review & Rating System

**Основна логіка:**

* Написання повноцінних текстових рецензій
* Ставлення оцінок (1–10)
* Можливість видаляти власні рецензії


### 4.3. Favorites Module

* Додавання фільмів у список обраного
* Перегляд власної колекції обраних фільмів
* Видалення зі списку
* Інтеграція з рекомендаційною моделлю


### 4.4. Social Module — Friends

**Користувацька взаємодія:**

* Пошук інших користувачів
* Надсилання та приймання запитів у друзі
* Перегляд профілю іншого користувача


### 4.5. Recommendation System

Система рекомендацій використовує поєднання:

* історії оцінок користувача
* жанрових уподобань
* фільмів, які подобаються друзям

Користувач отримує персоналізований список фільмів на головній сторінці.


### 4.6. Authentication & Authorization

**Механізми:**

* JWT-автентифікація
* Реєстрація та входження
* Захищені маршрути для всіх персоналізованих дій
* Рольова модель:

  * **User** — базовий функціонал
  * **Moderator** — модерація
  * **Admin** — управління контентом сайту + модерація


### 4.7. Administrator Panel

* Додавання нового контенту сайту
* Редагування та видалення існуючого
* Перегляд рецензій та можливість їх модерувати
* Перегляд користувачів
